# Temperature

A small, framework-agnostic toolkit for showing temperatures in the reader's
preferred unit. It ships two Svelte components — `Temperature` (a flash-free
display) and `TemperatureToggle` (a °C/°F switch) — plus the pure conversion,
locale-detection, storage and reactive-state helpers they are built on.

Nothing here is climate- or Reuters-specific: values go in as Celsius and render
as °C or °F based on one shared, persisted preference.

## Display a temperature

```svelte
<script>
  import { Temperature } from '@reuters-graphics/graphics-components';
</script>

<p>Today's high: <Temperature celsius={31.2} digits={1} /></p>
<p>Anomaly vs. normal: <Temperature celsius={3.4} digits={1} delta /></p>
```

`Temperature` takes a value in **degrees Celsius** and renders it in whichever
unit the reader prefers. Both the °C and °F strings are rendered and CSS shows
exactly one based on the `data-temp-unit` attribute on `<html>`, so even
server-rendered / prerendered markup never flashes the wrong unit and toggling
swaps every value on the page instantly with no per-node JavaScript.

Props: `celsius` (required), `delta` (render a signed difference like `+5°F`),
`digits` (decimal places, default `0`), `suffix` (append the degree suffix,
default `true`) and `degree` (include the degree ring, default `true`).

## Toggle the unit

```svelte
<script>
  import { TemperatureToggle } from '@reuters-graphics/graphics-components';
</script>

<TemperatureToggle />
```

`TemperatureToggle` flips the shared unit state, persists the explicit choice to
`localStorage` (mirrored to a cookie) and broadcasts a `temperature-unit-changed`
window event so every `Temperature`, chart and cross-bundle listener updates
together. With no props it drives the shared singleton; pass an explicit `state`
to control a scoped `TemperatureUnitState`.

## Flash-free rendering (the bootstrap)

For markup that is prerendered or served statically, add a tiny blocking script
to the document `<head>` so the correct `data-temp-unit` attribute is set
**before first paint**. `buildBootstrapScript()` returns that script as a string
you can inline in any framework:

```ts
import { buildBootstrapScript } from '@reuters-graphics/graphics-components';

const script = buildBootstrapScript(); // or pass { fahrenheitRegions, storageKey, ... }
```

Drop it into your app template's `<head>` (Astro layout, SvelteKit `app.html`,
Next `_document`, or plain HTML):

```astro
---
import { buildBootstrapScript } from '@reuters-graphics/graphics-components';
const bootstrap = buildBootstrapScript();
---

<head>
  <script is:inline set:html={bootstrap}></script>
</head>
```

The script resolves the unit (stored choice → cookie → locale/time-zone default
→ fallback), sets the `<html>` attribute and exposes
`window.getTemperatureUnit()` / `window.setTemperatureUnit()` for non-Svelte or
cross-bundle code. The ready-made default string is also exported as
`TEMPERATURE_UNIT_BOOTSTRAP`.

## First-visit default

A reader with no saved choice gets a locale-aware default: Fahrenheit for the
handful of countries that use it (the US, Liberia, and a few others) and Celsius
for the rest of the world.

```ts
import {
  resolveDefaultUnit,
  browserDefaultUnit,
} from '@reuters-graphics/graphics-components';

resolveDefaultUnit({ languages: ['en-US'] }); // 'fahrenheit'
resolveDefaultUnit({ languages: ['en-GB'] }); // 'celsius'
browserDefaultUnit(); // reads navigator + Intl (SSR-safe)
```

## Reactive state and helpers

`TemperatureUnitState` is a Svelte 5 rune-based store. Use the shared
`defaultUnitState` singleton, or provide a scoped instance with `setUnitContext`
/ `getUnitContext`. Pure helpers — `convertTemperature`, `convertDelta`,
`formatTemperature`, `formatDelta`, `unitSuffix`, `isTemperatureUnit`,
`otherUnit` — are exported for use in charts and other custom rendering.

```ts
import {
  formatTemperature,
  convertDelta,
} from '@reuters-graphics/graphics-components';

formatTemperature(22.4, 'fahrenheit'); // '72°F'
convertDelta(5, 'fahrenheit'); // 9  (deltas scale by 9/5, no +32 offset)
```

Every customizable name (storage key, cookie name, the `<html>` attribute and
the event) has a product-neutral default in `DEFAULT_CONFIG` and can be
overridden once via `resolveConfig` so the bootstrap, state and components all
agree.
