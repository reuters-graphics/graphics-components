# LineChart Component

A flexible, responsive line chart component built with D3.js and Svelte. Supports single and multiple series, small multiples layout, and extensive customization options.

## Features

- **Single & Multiple Series**: Display one or many data series on a single chart
- **Small Multiples**: Organize multiple chart groups for comparison
- **Responsive**: Automatically adapts to container width with customizable breakpoints
- **Y-Axis Formatting**: Flexible formatting with mode-based or custom approaches
- **Responsive Grid**: Intelligent grid layout for small multiples
- **Child Snippets**: Extend with custom SVG overlays and annotations
- **Legend Support**: Automatic legend generation with customizable colors
- **D3 Integration**: Leverages D3.js scales and utilities

## Installation

Import the component:

```svelte
import LineChart from '$lib/components/LineChart/LineChart.svelte';
```

## Basic Usage

### Single Series

```svelte
<script>
  const data = [
    { date: new Date('2025-01-01'), value: 100 },
    { date: new Date('2025-01-02'), value: 120 },
    { date: new Date('2025-01-03'), value: 110 }
  ];

  const series = [
    { key: 'value', label: 'Sales', color: '#1f77b4' }
  ];
</script>

<LineChart {data} {series} width={800} height={400} />
```

### Multiple Series

```svelte
<script>
  const data = [
    { date: new Date('2025-01-01'), revenue: 5000, expenses: 3000 },
    { date: new Date('2025-01-02'), revenue: 6000, expenses: 3500 }
  ];

  const series = [
    { key: 'revenue', label: 'Revenue', color: '#1f77b4' },
    { key: 'expenses', label: 'Expenses', color: '#ff7f0e' }
  ];
</script>

<LineChart {data} {series} width={800} height={400} />
```

## Layout Modes

### Single Chart

All series on one chart:

```svelte
<LineChart {data} {series} layout="single" />
```

### Small Multiples

Each series gets its own chart:

```svelte
<script>
  const chartGroups = [
    {
      groupId: 'revenue',
      title: 'Revenue',
      series: [{ key: 'revenue', label: 'Revenue' }]
    },
    {
      groupId: 'expenses',
      title: 'Expenses',
      series: [{ key: 'expenses', label: 'Expenses' }]
    }
  ];
</script>

<LineChart {data} chartGroups={chartGroups} layout="multiples" />
```

## Y-Axis Formatting

### Top-Only Mode (Default)

Format only the top tick:

```svelte
<LineChart
  {data}
  {series}
  yAxisConfig={{
    mode: 'top-only',
    suffix: ' K'
  }}
/>
```

### All-Ticks Mode

Format all tick labels:

```svelte
<LineChart
  {data}
  {series}
  yAxisConfig={{
    mode: 'all-ticks',
    prefix: '$',
    suffix: ''
  }}
/>
```

### Custom Formatter

Use a custom function for maximum control:

```svelte
<LineChart
  {data}
  {series}
  yValueFormatter={(value, context) => {
    if (context.isTopTick) {
      return `$${(value / 1000).toFixed(1)}K`;
    }
    return '';
  }}
/>
```

## Responsive Behavior

Enable responsive mode to automatically adjust layout based on container width:

```svelte
<script>
  const responsiveRules = [
    { breakpoint: 0, layout: 'single', columnsPerRow: 1 },
    { breakpoint: 600, layout: 'multiples', columnsPerRow: 1 },
    { breakpoint: 900, layout: 'multiples', columnsPerRow: 2 },
    { breakpoint: 1200, layout: 'multiples', columnsPerRow: 3 }
  ];
</script>

<LineChart
  {data}
  {series}
  responsive={true}
  responsiveRules={responsiveRules}
/>
```

## Props

### Main Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `data` | `Datum[]` | `[]` | Array of data points |
| `series` | `LineSeriesInput[]` | `[]` | Series configuration for single chart |
| `chartGroups` | `ChartGrouping[]` | `undefined` | Chart groupings for multiples layout |
| `layout` | `'single' \| 'multiples'` | `'single'` | Chart layout mode |
| `width` | `number` | `800` | Chart width in pixels |
| `height` | `number` | `400` | Chart height in pixels |
| `xKey` | `string` | `'date'` | Data key for X-axis |

### Y-Axis Formatting

| Prop | Type | Description |
|------|------|-------------|
| `yAxisConfig` | `YAxisConfig` | Y-axis configuration object |
| `yValueFormatter` | `function` | Custom formatter function |
| `yAxisFormat` | `string` | D3 format string |

### Display Options

| Prop | Type | Default |
|------|------|---------|
| `showGrid` | `boolean` | `true` |
| `showYAxis` | `boolean` | `true` |
| `showXAxis` | `boolean` | `true` |
| `showLegend` | `boolean` | `true` |

### Responsive

| Prop | Type | Description |
|------|------|-------------|
| `responsive` | `boolean` | Enable responsive mode |
| `responsiveRules` | `ResponsiveRule[]` | Breakpoint rules |

## Data Structure

### Datum

```typescript
interface Datum {
  [key: string]: string | number | Date;
}
```

### LineSeriesInput

```typescript
interface LineSeriesInput {
  key: string;              // Data key
  label?: string;           // Display label
  color?: string;           // Line color (hex)
  strokeWidth?: number;     // Line thickness (default: 2)
  showPoints?: boolean;     // Show data points
  showEndLabel?: boolean;   // Show value at line end
}
```

### ChartGrouping

```typescript
interface ChartGrouping {
  groupId: string;          // Unique identifier
  title?: string;           // Group title
  series: LineSeriesInput[]; // One or more series
}
```

## Performance Considerations

- Data should be sorted by X-axis value for optimal rendering
- For large datasets (>1000 points), consider data aggregation
- Use `showPoints={false}` for datasets with many points
- Responsive mode may add slight overhead; consider disabling if not needed

## Accessibility

- All axis labels are properly structured
- Legend provides semantic meaning for color-coded series
- SVG content is rendered in semantic order
- Consider adding ARIA labels for additional context
