# DataCard

A small grammar for building consistent **data cards** — compact panels that
pair a headline statistic with supporting labels, charts and footnotes. They
were built for the bottom-of-map overlays on the Reuters Climate Monitor, but
the pieces are presentational and work anywhere.

Compose a card from a `DataCard` shell and the grammar primitives:

- **`DataCard`** — the card shell (with an optional link mode). Each card is its
  own CSS size container, so its type scales to the card's own rendered width.
- **`DataCardTopline`** — the bold heading row.
- **`DataCardBigNumber`** — a large tabular-figure statistic with optional unit.
- **`DataCardLabel`** — a small label, optionally `muted` for context.
- **`DataCardColumns`** — a flex row for side-by-side value columns.
- **`DataCardColumn`** — a single value column inside `DataCardColumns`.
- **`DataCardChartWrap`** — a flex-growing wrapper for a chart.
- **`DataCardFootnote`** — small print pinned to the bottom of the card.
- **`DataCardRow`** — lays out several cards in a row that becomes a swipe
  carousel on mobile.

```svelte
<script>
  import {
    DataCard,
    DataCardTopline,
    DataCardBigNumber,
    DataCardLabel,
    DataCardColumns,
    DataCardColumn,
    DataCardFootnote,
  } from '@reuters-graphics/graphics-components';
</script>

<DataCard width="double">
  <DataCardTopline>Today's high</DataCardTopline>
  <DataCardColumns>
    <DataCardColumn>
      <DataCardBigNumber value="80" unit="°F" />
      <DataCardLabel>Forecast</DataCardLabel>
    </DataCardColumn>
    <DataCardColumn>
      <DataCardBigNumber value="+4.2" unit="°" color="#d64000" />
      <DataCardLabel color="#d64000">Above normal</DataCardLabel>
    </DataCardColumn>
  </DataCardColumns>
  <DataCardFootnote>* Calculated in UTC for comparison</DataCardFootnote>
</DataCard>
```

## A single card

The simplest card pairs a topline with one headline statistic.

<Canvas of={DataCardStories.Card} />

Each card is a CSS size container, so it **fills the width it's given** (up to
`max-width`) rather than shrink-wrapping its content — that's what lets cards
line up cleanly in a `DataCardRow`. So rather than wrapping a lone stat in a
sizing `div`, pick a named width.

## Width

A card takes a named **`width`** prop with two sizes. It sets the card's
`max-width`, and because the card is a size container its type scales to match:

        <code>width</code>
      </th>
      <th>max-width</th>
      <th>use</th>

  </thead>
  <tbody>
    <tr>
      <td>
        <code>single</code>
      </td>

      <td>one-number cards (the default)</td>
    </tr>
    <tr>
      <td>
        <code>double</code>
      </td>
      <td>230px</td>
      <td>two-column cards and charts</td>
    </tr>
  </tbody>

```svelte
<DataCard width="double">…</DataCard>
```

Need a one-off size? Override either with the `--dc-card-max-width` CSS
variable — it always wins over the named width.

```svelte
<DataCard style="--dc-card-max-width: 280px;">…</DataCard>
```

## A row of cards

`DataCardRow` lays cards side by side and gives them a shared, definite width so
their type scales consistently. Pass `maxWidth` to cap and center the row. When
every card fits, they sit centered with no scrolling:

<Canvas of={DataCardStories.Row} />

The same row turns into a horizontal swipe carousel **only when the cards don't
all fit** — there's no separate "carousel mode" to opt into. Overflow is measured
from the row's own width, so it works whether the row is narrowed by a small
viewport, a Pym iframe or a constrained parent. When it overflows it scrolls with
scroll-snap (and scroll-chaining guards, so a fling at the first/last card
doesn't nudge the surrounding page sideways); when there's room for every card it
stays the plain centered row above:

<Canvas of={DataCardStories.Carousel} />

## Two columns

Wrap each value in a `DataCardColumn` and group them in `DataCardColumns` to
place two (or more) columns side by side inside a single card — handy for a
value and its comparison. Each `DataCardColumn` flexes to share the row evenly,
so there's no need to hand-roll `flex` styles.

## Theming

Cards read Reuters theme tokens (`--theme-colour-text-primary`,
`--theme-colour-text-secondary`, `--theme-colour-brand-rules`,
`--theme-font-family-sans-serif`) so they pick up a surrounding `Theme`
automatically, falling back to sensible defaults otherwise. Fine-grained sizing
can be overridden with the `--dc-*` custom properties below. Set them on a card
(or any ancestor — they inherit) to tune a single instance or a whole group.
